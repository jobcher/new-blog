#### Linux 内核迎来多内核架构提案
<p><b>Linux 内核迎来多内核架构提案</b><br><br>9 月 21 日消息，Multikernel Technologies 向 Linux 内核邮件列表提交了一份 多内核架构（Multikernel Architecture） 的 RFC 补丁，并已开源相关代码。该方案允许多个独立的内核实例在同一物理机上并存运行，每个内核可绑定特定 CPU 核心，同时共享底层硬件资源。<br><br>开发者 Cong Wang 表示，多内核架构的优势包括：<br><br>更强的故障隔离与安全性；<br><br>高于传统虚拟机的资源利用率；<br><br>通过 KHO（Kernel Hand Over） 实现潜在的零停机内核更新；<br><br>支持实时内核与通用内核并行运行，适配安全关键或专用任务。<br><br>该实现基于 kexec 基础设施 管理多个内核镜像，并通过专用的 IPI 框架实现跨内核通信。<br><br>目前社区已有开发者表示，曾在 x86 硬件上尝试过类似概念，但要突破实验阶段仍面临不少复杂性。Linux 上游是否采纳这一提案，还有待观察。<br><br><a href="https://www.phoronix.com/news/Linux-Multi-Kernel-Patches" target="_blank" rel="noopener" onclick="return confirm('Open this link?\n\n'+this.href);">phoronix</a></p>
