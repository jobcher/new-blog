#### 重要/漏洞：NGINX Rift / NGINX ngx_http_rewrite_module 堆缓冲区存在溢出漏洞
<p><b>重要/漏洞：NGINX Rift / NGINX ngx_http_rewrite_module 堆缓冲区存在溢出漏洞</b><br><br><b>漏洞编号：</b>CVE-2026-42945<br><b>重要等级：</b>重要[需要关注的]<br><b>CVSS 分数：</b>CVSS v4.0：9.2 Critical；CVSS v3.1：8.1 High<br><b>注：</b>NGINX 官方安全页将该项标注为 medium，但 F5/CNA 与 NVD 记录显示其在特定条件下可导致高危影响。<br><br><b>影响范围：</b><br></p><blockquote>NGINX Open Source 0.6.27 至 1.30.0 受影响，1.30.1 与 1.31.0 及以上版本不受影响。</blockquote><br><blockquote>NGINX Plus R32 至 R36 也在公开影响范围内；部分基于 NGINX 的 F5/NGINX 产品，如 NGINX Instance Manager、F5 WAF for NGINX、NGINX App Protect、NGINX Gateway Fabric、NGINX Ingress Controller 等也被列入影响范围。</blockquote><br><br><b>触发条件：</b><br>受影响版本的 NGINX 配置中同时存在以下模式时风险较高：<br><blockquote>1. 使用 rewrite 指令；<br>2. rewrite 的替换字符串中包含问号 ?；<br>3. 后续同一作用域内跟随 rewrite、if 或 set 指令；<br>4. 使用未命名 PCRE 捕获组变量，如 $1、$2。</blockquote><br><br><b>漏洞原理：</b><br><blockquote>该漏洞源于 NGINX ngx_http_rewrite_module 中脚本引擎的状态传递不一致。当 rewrite 的 replacement 中包含 ?
